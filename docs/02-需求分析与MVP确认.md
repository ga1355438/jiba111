> 本文档是 PRD 之前的收束动作，不是正式 PRD。
> 采用"课堂替代调研"方式：基于校园图书馆场景合理假设 + 同类系统常见能力 + 课堂工作量收束。
>
> 输入：docs/01-项目立项单.md（已确认）

## 1. 目标用户与核心场景

### 1.1 用户角色

| 角色 | 画像 | 使用频率 |
|------|------|----------|
| 学生（主用户） | 需要到图书馆自习的在校学生，习惯提前占座 | 每天 1-2 次 |
| 图书馆管理员 | 负责座位资源管理，了解各时段座位使用情况 | 每天数次 |

### 1.2 核心场景

**场景 A — 学生预约座位（主线）**

> 小明下午想自习，打开系统看看三楼还有没有带电源的座位，选了一个 14:00-16:00 的时段，提交预约。之后在"我的预约"里能看到这条记录，临时有事可以取消。
>
> 异常方向：如果选的时段已被预约，系统提示"该时段已被预约"，引导换时段。

**场景 B — 管理员管理资源（主线）**

> 管理员登录后台，看到今天一共被预约了多少个座位，哪些座位比较热门，有没有需要维护下线的座位。新增了一排新到的座位，把坏掉的座位标记为维护中。
>
> 异常方向：如果新增座位编号重复，系统提示"座位编号已存在"；如果删除有预约记录的座位，系统阻止并提示。

**场景 C — 查看预约统计（支线）**

> 管理员想了解近期座位使用趋势，打开统计页看到总预约数、各时段使用占比、热门座位排行。
>
> 异常方向：无预约数据时，统计页显示"暂无数据"，不报错。

## 2. 用户最关心的问题

| 排序 | 学生关心的问题 | 管理员关心的问题 |
|------|---------------|-----------------|
| 1 | 这个座位现在能不能约？ | 今天一共被约了多少次？ |
| 2 | 有没有带电源的座位？ | 哪些座位被用得最多？ |
| 3 | 我约了哪些时段？能取消吗？ | 座位信息能不能随时增删改？ |
| 4 | 操作快不快，是不是一步能约上？ | 有没有人占座不来？ |

> 以上问题来自对校园图书馆场景的合理假设，非真实用户访谈。

### 2.1 需求追溯：用户问题 → 功能映射

| 用户问题 | 对应功能 | 实现方式 |
|----------|----------|----------|
| 这个座位现在能不能约？ | F02 座位列表 + F03 座位详情 | 列表展示状态字段，详情展示各时段预约状态 |
| 有没有带电源的座位？ | F02 座位列表 | HasPower 字段展示"有/无" |
| 我约了哪些时段？能取消吗？ | F05 我的预约 + F06 取消预约 | 按账号筛选列表，Status=0 显示取消按钮 |
| 操作快不快，一步能约上？ | F04 预约提交 | 详情页直接跳预约表单，减少中间步骤 |
| 今天一共被约了多少次？ | F11 统计页 | 实时 COUNT 查询，首页展示 |
| 哪些座位被用得最多？ | F11 统计页 | 按 SeatId 分组 Count，TOP5 排序 |
| 座位信息能不能随时增删改？ | F09 座位管理 | 管理端 CRUD 表单 |
| 有没有人占座不来？ | （本期不做） | 需签到功能，立项单已排除 |

### 关于"座位详情"时段状态的口径统一

立项单原文"含所选日期各时段预约状态"存在一个歧义：座位详情页是否需要用户先选择日期才能看到时段状态？

**结论：简化处理。** 座位详情页默认展示**当天**各时段的预约状态，不需要用户选择日期。理由：
1. 减少一次交互，操作更快
2. 课堂项目不需要跨日查询
3. 如需查看其他日期，在"我的预约"中按日期筛选即可

> 此口径将在 PRD Lite 中进一步明确。

## 8. 进入 PRD Lite 前的最终口径

| 项目 | 口径 |
|------|------|
| 页面总数 | 10 个视图页面 |
| 数据表 | 3 张（Seat、Reservation、AdminUser） |
| 功能总数 | MVP 12 项 + 暂缓 3 项 |
| 用户端功能 | 8 项（首页、座位列表、座位详情、预约提交、我的预约、取消预约、体验账号切换、统一布局） |
| 管理端功能 | 4 项（登录、座位管理、预约管理、统计页） |
| 时段 | 固定 5 个时段，不可自定义 |
| 体验账号 | 3 个学生 + 1 个管理员 |
| 座位详情默认日期 | 当天，不支持选择日期 |
| 预约冲突校验 | 同一座位 + 同一日期 + 同一时段不可重复预约 |
| 取消预约规则 | 仅 Status=0（已预约）可取消，改为 Status=2（已取消） |
| 统计页范围 | 纯数据展示，不做图表可视化 |
| 分页 | 座位列表每页 10 条 |
| 体验账号切换方式 | 用户端顶部导航栏下拉框，切换后所有页面按新账号筛选 |
