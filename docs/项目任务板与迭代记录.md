## 1. 任务总览

### 1.1 四列状态定义

| 状态 | 含义 | 进入条件 |
|------|------|----------|
| 待开发 | 任务已识别，尚未开始 | Sprint 规划确认后 |
| 进行中 | 正在编码或调试 | 开发者认领后 |
| 待验证 | 代码完成，等待功能验证 | 编译通过后 |
| 已完成 | 功能验证通过 | 验收标准全部满足 |

### 1.2 任务编号规则

```
T{Sprint编号}-{序号}

Sprint 0 → T12-01, T12-02, ...
Sprint 1 → T13-01, T13-02, ...
Sprint 2 → T14-01, T14-02, ...
Sprint 3 → T15-01, T15-02, ...
Sprint 4 → T16-01, T16-02, ...
```
