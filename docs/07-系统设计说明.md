## 1. 系统设计目标

### 1.1 核心目标

| 目标 | 说明 |
|------|------|
| **代码可维护** | 分层清晰，改一处不影响其他层 |
| **职责可追溯** | 每个功能知道代码在哪、谁负责 |
| **接口可对接** | 前端页面和后端逻辑能顺利联调 |
| **扩展可预期** | 后续加功能有固定位置放代码 |

### 1.2 设计原则

| 原则 | 落地方式 |
|------|----------|
| 单一职责 | Controller 接收请求并处理业务逻辑，DataAccess 只管数据访问 |
| 依赖注入 | 通过 ASP.NET Core 内置 DI 容器管理依赖 |
| 约定优于配置 | 遵循 ASP.NET Core 默认约定（路由、命名、文件位置） |
| 最小暴露 | 页面只看到它需要的数据（ViewModel），不暴露 Entity 全貌 |

## 4. 分层说明

### 4.1 Controller 层（控制器）

**职责**：接收 HTTP 请求，调用业务逻辑，返回视图或重定向。

**位置**：`Controllers/` 目录

**规则**：
- 每个 Controller 对应一个功能模块
- Controller 只做三件事：① 接收参数 ② 调用 Service/DataAccess ③ 返回结果
- 不在 Controller 里写业务逻辑（如冲突校验、统计计算）
- 不在 Controller 里直接操作数据库（通过 EF Core 上下文间接操作）

**代码示例**：
```csharp
// ReservationController.cs
public class ReservationController : Controller
{
    // GET: /Reservation/Create/{seatId}
    public IActionResult Create(int seatId)
    {
        // 1. 从数据库查询座位信息
        // 2. 构建 ViewModel
        // 3. 返回视图
    }

    // POST: /Reservation/Create
    [HttpPost]
    public IActionResult Create(ReservationCreateViewModel model)
    {
        // 1. 校验冲突
        // 2. 写入数据库
        // 3. 重定向到我的预约页
    }
}
```

### 4.2 Service 层（业务逻辑）（本期不使用）

**职责**：封装业务规则，供 Controller 调用。

**位置**：`Services/` 目录（本期不使用，业务逻辑直接写在 Controller 中）

**规则**：
- 业务规则放这里（如：冲突校验、统计计算、权限判断）
- 不依赖 Controller 或 View
- 可被多个 Controller 复用

**本期做法**：由于项目较小，业务逻辑直接写在 Controller 中（如冲突校验、统计计算），避免过度设计。如后续功能增加，再抽取 Service 层。

### 4.3 DataAccess 层（数据访问）

**职责**：封装数据库操作，供 Controller 或 Service 调用。

**位置**：`Data/` 目录

**规则**：
- 通过 EF Core DbContext 操作数据库
- 种子数据放在 `Data/SeedData.cs`
- 数据库迁移使用 `dotnet ef` 命令

**代码示例**：
```csharp
// Data/AppDbContext.cs
public class AppDbContext : DbContext
{
    public DbSet<Seat> Seats { get; set; }
    public DbSet<Reservation> Reservations { get; set; }
    public DbSet<AdminUser> AdminUsers { get; set; }
}
```

### 4.4 ViewModel 层（视图模型）（本期使用 ViewBag 替代）

**职责**：定义每个页面需要的数据结构，作为 Controller → View 的桥梁。

**位置**：`Models/ViewModels/` 目录（本期不使用独立 ViewModel 类，使用 ViewBag 传递数据）

**规则**：
- 每个页面一个 ViewModel（或共用）
- 只包含页面需要的字段，不暴露 Entity 全貌
- 命名格式：`{页面名}ViewModel`

**本期做法**：由于项目较小，使用 ViewBag 替代独立 ViewModel 类，简化开发。如后续功能增加，再抽取 ViewModel。

**ViewModel 参考**：

| ViewModel | 对应页面 | 主要字段 |
|-----------|----------|----------|
| HomeViewModel | 用户首页 | CurrentUserName, Today, TotalSeats, AvailableSeats |
| SeatListViewModel | 座位列表页 | Seats (List), CurrentPage, TotalPages |
| SeatDetailViewModel | 座位详情页 | Seat, ReservedSlots, Today |
| ReservationCreateViewModel | 预约提交页 | Seat, UserName, ReserveDate, TimeSlot |
| MyReservationViewModel | 我的预约页 | Reservations (List), CurrentUserName |
| AdminSeatViewModel | 座位管理页 | Seats (List) |
| AdminReservationViewModel | 预约管理页 | Reservations (List), FilterDate, FilterStatus |
| StatisticsViewModel | 统计页 | TotalCount, TodayCount, ActiveCount, TopSeats, TimeSlotDistribution |

### 4.5 Entity 层（实体）

**职责**：定义数据库表结构，与数据库一一对应。

**位置**：`Models/Entities/` 目录

**规则**：
- 每个 Entity 对应一张数据库表
- 使用 Data Annotations 定义字段约束
- 命名格式：单数名词（Seat、Reservation、AdminUser）

**代码示例**：
```csharp
// Models/Entities/Seat.cs
public class Seat
{
    public int Id { get; set; }
    public string Name { get; set; }
    public string Location { get; set; }
    public bool HasPower { get; set; }
    public int Status { get; set; }
    public DateTime CreatedAt { get; set; }
}
```

## 7. 主链路工程落点

### 7.1 用户预约流程（核心闭环）

```
用户首页 ──→ 座位列表页 ──→ 座位详情页 ──→ 预约提交页 ──→ 我的预约页
   │              │              │              │              │
   ▼              ▼              ▼              ▼              ▼
HomeController  SeatController  SeatController  ReservationController  ReservationController
.Index()        .List()         .Detail()       .Create(GET)           .My()
                                  │              │
                                  │              ▼
                                  │         ReservationController
                                  │         .Create(POST)
                                  │         ├── 校验冲突
                                  │         ├── 写入数据库
                                  │         └── 重定向到 .My()
                                  │
                                  └── 查询时段状态
                                      ├── 查询 Seat 表
                                      └── 查询 Reservation 表（当日该座位）
```

**关键落点**：

| 步骤 | 代码位置 | 关键逻辑 |
|------|----------|----------|
| 查询座位列表 | `SeatController.List()` | 分页查询 Seat 表 |
| 查询座位详情 | `SeatController.Detail()` | 查询单个 Seat + 当日 Reservation |
| 提交预约 | `ReservationController.Create(POST)` | 冲突校验 + 写入 Reservation |
| 查询我的预约 | `ReservationController.My()` | 按 UserName 筛选 Reservation |

### 7.2 用户取消流程

```
我的预约页 ──→ 点击取消 ──→ 确认弹窗 ──→ POST Cancel ──→ 刷新页面
   │                          │              │
   ▼                          ▼              ▼
ReservationController    前端 JS         ReservationController
.My()                    confirm()       .Cancel()
                                             │
                                             ▼
                                         校验权限
                                         ├── 只能取消自己的
                                         ├── 只能取消 Status=0
                                         └── 更新 Status=2
```

**关键落点**：

| 步骤 | 代码位置 | 关键逻辑 |
|------|----------|----------|
| 取消预约 | `ReservationController.Cancel()` | 校验权限 + 更新 Status |

### 7.3 管理员登录流程

```
登录页 ──→ 输入账号密码 ──→ POST Login ──→ 校验 ──→ 成功: 跳转座位管理
   │                          │              │
   ▼                          ▼              ▼
AdminController          AdminController   校验逻辑
.Login(GET)              .Login(POST)      ├── 查询 AdminUser 表
                                           ├── PasswordHasher 比对密码
                                           └── 写入 Session
```

**关键落点**：

| 步骤 | 代码位置 | 关键逻辑 |
|------|----------|----------|
| 管理员登录 | `AdminController.Login(POST)` | 查询账号 + PasswordHasher 比对 + 写入 Session |
| 权限检查 | AdminController 构造函数或 Action Filter | 未登录跳转登录页 |

### 7.4 管理员座位管理流程

```
座位管理页 ──→ 新增/编辑/删除 ──→ 表单提交 ──→ 校验 ──→ 刷新列表
   │                          │              │
   ▼                          ▼              ▼
AdminController          AdminController   校验逻辑
.SeatIndex()             .SeatCreate/      ├── 新增: 校验 Name 唯一
                       .SeatEdit/         ├── 编辑: 校验 Name 唯一
                       .SeatDelete()      └── 删除: 校验无关联预约
```

**关键落点**：

| 步骤 | 代码位置 | 关键逻辑 |
|------|----------|----------|
| 座位列表 | `AdminController.SeatIndex()` | 查询所有 Seat |
| 新增座位 | `AdminController.SeatCreate(POST)` | 校验 Name 唯一 + 写入 Seat |
| 编辑座位 | `AdminController.SeatEdit(POST)` | 校验 Name 唯一 + 更新 Seat |
| 删除座位 | `AdminController.SeatDelete(POST)` | 校验无关联预约 + 删除 Seat |

## 8. 异常处理原则

### 8.1 异常分类

| 异常类型 | 处理方式 | 示例 |
|----------|----------|------|
| 数据校验失败 | 返回视图 + ViewBag.Error | 冲突校验、编号重复 |
| 权限不足 | 重定向到登录页 | 未登录访问管理页 |
| 资源不存在 | 返回 404 或首页 | 座位 ID 无效 |
| 数据库异常 | 返回错误页面 | 数据库连接失败 |
| 未捕获异常 | 全局异常处理 | 代码 Bug |

### 8.2 错误信息传递方式

| 方式 | 适用场景 | 代码示例 |
|------|----------|----------|
| ViewBag.Error | 表单提交失败 | `ViewBag.Error = "该时段已被预约";` |
| TempData | 一次性提示 | `TempData["Success"] = "操作成功";` |
| 状态码 | 资源不存在 | `return NotFound();` |
| Error 视图 | 系统异常 | `return View("Error");` |

**异常处理代码示例**：
```csharp
// ReservationController.cs
[HttpPost]
public async Task<IActionResult> Create(Reservation model)
{
    try
    {
        // 业务逻辑...
        await _context.SaveChangesAsync();
        return RedirectToAction(nameof(My));
    }
    catch (Exception)
    {
        ViewBag.Error = "提交失败，请稍后重试";
        return View(model);
    }
}
```

### 8.3 异常处理位置

```
Controller Action
    │
    ├── try-catch 包裹数据库操作
    │
    ├── 校验失败 → ViewBag.Error → 返回视图
    │
    └── 未捕获异常 → 全局异常处理 → Error 视图
```

## 9. Session / Cookie 使用原则

### 9.1 Session 使用场景

| Session Key | 类型 | 说明 | 超时 |
|-------------|------|------|------|
| CurrentUserName | string | 当前体验账号（默认"张三"，通过 `?? "张三"` 实现） | 30 分钟 |
| AdminUsername | string | 管理员登录名 | 30 分钟 |

### 9.2 Session 操作位置

| 操作 | 代码位置 | 说明 |
|------|----------|------|
| 读取体验账号 | Controller Action | `Context.Session.GetString("CurrentUserName")` |
| 切换体验账号 | `ReservationController.SwitchUser()` | 写入 Session 并刷新页面 |
| 管理员登录 | `AdminController.Login()` | 写入 AdminUsername |
| 管理员登出 | `AdminController.Logout()` | 清除 Session |
| 权限检查 | AdminController 使用 `IsLoggedIn()` 私有方法 | 读取 Session，为空则重定向 |

### 9.3 Session 配置

```csharp
// Program.cs
builder.Services.AddSession(options =>
{
    options.IdleTimeout = TimeSpan.FromMinutes(30);
    options.Cookie.HttpOnly = true;
    options.Cookie.IsEssential = true;
});

app.UseSession();
```

### 9.4 Cookie 使用

本期不直接使用 Cookie，通过 Session 间接管理。Session 底层依赖 Cookie 存储 SessionId。
