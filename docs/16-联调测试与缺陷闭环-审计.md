## B. 发现的问题清单

### B-1. 统计页 RuntimeBinderException 崩溃（P0-阻塞）

**文件**：`Views/Admin/Statistics.cshtml:42`

```csharp
@if (((List<dynamic>)stats.TopSeats).Any())
```

**错误**：`RuntimeBinderException: Cannot convert type 'System.Collections.Generic.List<<>f__AnonymousType1<int,int>>' to 'System.Collections.Generic.List<object>'`

**根因**：AdminController 构造 `stats.TopSeats` 为匿名类型列表，Razor 运行时绑定无法将 `List<anonymous>` 转为 `List<dynamic>`。

**影响**：管理端统计页完全不可用，返回 500 错误。

**修复方案**：将 `List<dynamic>` 改为 `IEnumerable<dynamic>` 或使用 `@foreach` 直接遍历 `stats.TopSeats` 而不做强制类型转换。

---

### B-2. 测试 TC-01/TC-05 Strict Mode Violation（P1）

**现象**：`text=当前用户` 匹配到 2 个元素（导航栏 + 页面内容），Playwright strict mode 报错。

**根因**：测试选择器不够精确，导航栏和页面正文都包含"当前用户"文本。

**影响**：2 个测试用例失败，但不影响功能。

**修复方案**：使用 `.first()` 或更精确的选择器（如 `p.text-muted`）。

---

### B-3. 测试 TC-11 重复预约冲突未触发（P1）

**现象**：第二次提交预约后直接跳转到"我的预约"页，未显示"该时段已被预约"错误。

**根因**：第一次预约创建后，第二次提交可能使用了不同的时间段（radio button 默认选中第一个可用时段），或者 Session 状态导致冲突检测绕过。

**影响**：冲突校验测试未通过，需排查是测试问题还是业务逻辑问题。
