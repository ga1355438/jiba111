## 1. 代码规范

### 1.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| Controller | `{功能}Controller` | `SeatController`, `AdminController` |
| Entity | 单数名词，PascalCase | `Seat`, `Reservation`, `AdminUser` |
| DbSet | 复数名词 | `Seats`, `Reservations`, `AdminUsers` |
| Action 方法 | 动词或动词+名词 | `List()`, `Detail()`, `Create()` |
| 变量 | camelCase | `seatList`, `reservationCount` |
| 常量 | PascalCase | `TimeSlots.Morning1` |
| 枚举 | PascalCase | `SeatStatus.Available` |

### 1.2 Controller 规范

```
Controller 只做三件事：
1. 接收参数（Action 方法入参）
2. 调用逻辑（EF Core 查询/写入）
3. 返回结果（View / RedirectToAction / NotFound）

不在 Controller 中：
- 编写复杂业务逻辑（应抽取 Service）
- 直接操作 SQL（应使用 EF Core）
- 硬编码字符串（应使用常量或配置）
```

### 1.3 数据访问规范

```
查询规范：
- 使用 async/await 异步查询
- 使用 AsNoTracking() 优化只读查询
- Include() 明确加载关联数据
- 分页查询使用 Skip().Take()

写入规范：
- 先校验后写入
- 使用 SaveChangesAsync()
- 异常时返回 ViewBag.Error
```

## 3. View 规范

### 3.1 View 布局

```
所有页面使用 _Layout.cshtml 布局：
- 用户端：导航栏（含体验账号切换）
- 管理端：导航栏（含登录/登出）
- 页脚：版权信息
```

### 3.2 数据传递

```
本期使用 ViewBag 传递数据：
- ViewBag.Error = "错误信息"
- ViewBag.TotalCount = 100
- ViewBag.CurrentPage = 1

不使用 ViewModel（简化开发）
```

### 3.3 表单规范

```
表单提交：
- 使用 @Html.AntiForgeryToken()
- 使用 asp-for 绑定模型
- 错误信息通过 ViewBag.Error 显示

示例：
<form asp-action="Create" method="post">
    @Html.AntiForgeryToken()
    <div asp-validation-summary="All" class="text-danger"></div>
    ...
</form>
```

## 4. 数据库规范

### 4.1 迁移策略

```
使用 EF Core Migrations：
- 开发环境：dotnet ef migrations add {MigrationName}
- 生产环境：app.Database.Migrate()
- 种子数据：SeedData.Initialize()
```

### 4.2 查询规范

```csharp
// 正确示例
var seats = await _context.Seats
    .AsNoTracking()
    .OrderBy(s => s.Id)
    .ToListAsync();

// 正确示例（带分页）
var reservations = await _context.Reservations
    .Include(r => r.Seat)
    .AsNoTracking()
    .OrderByDescending(r => r.ReserveDate)
    .Skip((page - 1) * pageSize)
    .Take(pageSize)
    .ToListAsync();
```

## 5. 安全规范

### 5.1 认证授权

```
管理端：
- 使用 AdminAuthorizationFilter 检查登录状态
- Login/Logout 使用 [AllowAnonymous]
- 其他 Action 自动要求登录

用户端：
- 体验账号通过 Session 管理
- 默认账号"张三"
```

### 5.2 密码存储

```
使用 PasswordHasher：
- 存储：PasswordHelper.HashPassword(password)
- 验证：PasswordHelper.VerifyPassword(password, hash)
- 不使用 MD5
```

### 5.3 CSRF 防护

```
所有 POST 操作使用：
- @Html.AntiForgeryToken()
- [ValidateAntiForgeryToken]
```

## 6. Git 规范

### 6.1 分支策略

```
main        - 生产分支
develop     - 开发分支
feature/*   - 功能分支
```

### 6.2 提交信息

```
格式：{类型}: {描述}

类型：
- feat: 新功能
- fix: 修复
- docs: 文档
- style: 格式
- refactor: 重构
- test: 测试
- chore: 构建/工具

示例：
feat: 实现座位列表分页
fix: 修复预约冲突校验
docs: 更新数据库设计文档
```
