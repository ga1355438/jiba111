## 2. 全局布局（_Layout.cshtml）

### 导航栏结构

```
[图书馆座位预约]  首页  座位列表  我的预约  |  [张三 ▼]    管理员登录
                                  切换账号 →  李四
                                              王五
```

**用户端导航栏**（管理员未登录时）：
- 左侧：系统名称（链接首页）
- 中间：首页、座位列表、我的预约
- 右侧：体验账号下拉框（张三/李四/王五）+ 管理员登录链接

**管理端导航栏**（管理员已登录时）：
- 左侧：系统名称
- 中间：首页、座位列表、我的预约
- 右侧：管理员名 + 座位管理、预约管理、统计页、退出登录

### 体验账号切换

| 项目 | 内容 |
|------|------|
| 位置 | 导航栏右侧下拉框 |
| 选项 | 张三、李四、王五 |
| 默认 | 张三 |
| 行为 | 切换后 POST 到 `/Reservation/SwitchUser`，Session 更新，刷新当前页 |
| 影响范围 | 所有页面的"当前用户"显示和"我的预约"数据筛选 |

### 页面 4：预约提交

**路由**：`/Reservation/Create/{seatId}`（GET 展示表单，POST 提交）

**页面结构**：
```
┌─────────────────────────────────┐
│ 预约座位                          │
│                                   │
│ 座位编号：C-04                     │
│ 位置：三楼靠窗                     │
│ 当前用户：张三                     │
│                                   │
│ ┌─────────────────────────────┐  │
│ │ 预约日期：[2026-07-14     ] │  │
│ │ 时    段：[14:00-16:00 ▼]  │  │
│ └─────────────────────────────┘  │
│                                   │
│ [提交预约]  [返回]                │
└─────────────────────────────────┘
```

**冲突时**：
```
┌─────────────────────────────────┐
│ ⚠ 该时段已被预约，请选择其他时段   │
│ ...（表单保留已填内容）            │
└─────────────────────────────────┘
```

**字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 座位编号 | 只读文本 | 从 URL 参数读取 |
| 位置 | 只读文本 | 从 Seat 表读取 |
| 当前用户 | 只读文本 | Session["CurrentUserName"] |
| 预约日期 | date 输入框 | 默认今天，min=今天 |
| 时段 | 下拉选择 | 5 个固定选项 |

**时段选项**：
1. 08:00-10:00
2. 10:00-12:00
3. 14:00-16:00
4. 16:00-18:00
5. 19:00-21:00

**校验规则**：

| 校验 | 触发条件 | 提示信息 |
|------|----------|----------|
| 日期为空 | 前端 | 请选择预约日期 |
| 时段为空 | 前端 | 请选择时段 |
| 时段冲突 | 后端 | 该时段已被预约，请选择其他时段 |
| 座位不存在 | 后端 | 404 |
| 座位维护中 | 后端 | 404 |

**跳转**：
- 提交成功 → `/Reservation/My`
- 提交冲突 → 当前页（保留表单内容，显示错误提示）
- 点击"返回" → `/Seat/Detail/{seatId}`

## 6. 数据流说明

### 体验账号切换

```
导航栏下拉框选择 → POST /Reservation/SwitchUser
→ Session["CurrentUserName"] = 选择的姓名
→ 刷新当前页
→ 所有页面读取 Session["CurrentUserName"] 作为筛选条件
```

### 预约提交

```
POST /Reservation/Create/{seatId}
→ 读取表单：ReserveDate, TimeSlot
→ 读取 Session["CurrentUserName"] 作为 UserName
→ 查询冲突：SeatId + ReserveDate + TimeSlot + Status=0
→ 有冲突 → 返回错误
→ 无冲突 → 写入 Reservation 表（Status=0）→ 跳转 /Reservation/My
```

### 取消预约

```
POST /Reservation/Cancel/{id}
→ 校验：UserName == Session["CurrentUserName"]
→ 校验：Status == 0
→ 通过 → Status 改为 2 → 跳转 /Reservation/My
→ 不通过 → 跳转 /Reservation/My
```

### 管理员登录

```
POST /Admin/Login
→ 校验 Username + MD5(Password) 匹配 AdminUser 表
→ 成功 → Session["AdminUsername"] = Username → 跳转 /Admin/Seat/Index
→ 失败 → 返回错误提示
```
