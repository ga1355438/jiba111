## B. 发现的问题清单

### 问题 1：座位新增/编辑表单页未计入页面总数

| 项目 | 说明 |
|------|------|
| 位置 | 第1节 辅助路由表格 |
| 描述 | 座位新增（`/Admin/SeatCreate`）和座位编辑（`/Admin/SeatEdit/{id}`）标注为"表单页"，但未计入9页总数 |
| 影响 | 轻微，实际开发时需要2个额外页面，但作为表单弹窗/内嵌处理也可行 |
| 建议 | 明确说明这两个页面是独立页面还是弹窗/内嵌方式 |

### 问题 2：管理员登出路由 HTTP 方法标注不一致

| 项目 | 说明 |
|------|------|
| 位置 | 第1节 辅助路由表格 |
| 描述 | 管理员登出标注为 `GET`，但按REST规范和安全性应为 `POST` |
| 影响 | 轻微，GET登出存在CSRF风险 |
| 建议 | 改为 `POST` 方法 |

### 问题 3：页面跳转矩阵缺少部分路径

| 项目 | 说明 |
|------|------|
| 位置 | 第7.2节 页面跳转矩阵 |
| 描述 | 缺少以下跳转路径：座位管理页→预约管理页、座位管理页→统计页、预约管理页→座位管理页、统计页→座位管理页 |
| 影响 | 轻微，管理端导航栏已覆盖这些跳转 |
| 建议 | 补充管理端页面间的导航跳转 |

### 问题 4：预约状态"已过期"缺少触发机制说明

| 项目 | 说明 |
|------|------|
| 位置 | 第6.2节 预约状态 |
| 描述 | 状态3（已过期）标注为"本期可选"，但未说明触发机制（自动过期还是手动标记） |
| 影响 | 轻微，不影响核心流程 |
| 建议 | 明确过期机制或标注"本期不做" |

### 问题 5：体验账号切换后页面刷新目标不明确

| 项目 | 说明 |
|------|------|
| 位置 | 第4节 用户主链路3 |
| 描述 | 切换体验账号后"当前页面刷新"，但从座位详情页切换后应该回到列表页更合理 |
| 影响 | 轻微，用户体验可优化 |
| 建议 | 明确切换后统一跳转到座位列表页 |

## E. 是否可以进入下一步"页面卡与 UI 规范"

**可以进入下一步。**

理由：
1. **页面总数可控**：9页（+2个表单页可选）符合课堂项目规模
2. **功能完整**：用户端5页覆盖预约全流程，管理端4页覆盖管理需求
3. **主链路闭环**：预约流程、取消流程、管理流程均完整
4. **状态流转合理**：座位状态、预约状态、会话状态定义清晰
5. **无缺页重页**：每个页面职责明确，无功能重叠

**建议**：在进入下一步前，快速修复C节中的2个必改项（明确表单页归属、登出改POST）。
