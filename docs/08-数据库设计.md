# 数据库设计

> 文档类型：数据库表结构、关系、索引、种子数据设计
> 输入来源：03-PRD-Lite.md、04-页面树与业务流程.md、07-系统设计说明.md

## 1. 数据设计目标

| 目标 | 说明 |
|------|------|
| **业务支撑** | 支撑座位浏览、预约提交、冲突校验、取消预约、统计查询等核心业务 |
| **数据一致性** | 同一座位同一日期同一时段不能被重复有效预约 |
| **查询效率** | 通过索引优化高频查询场景（冲突校验、我的预约、统计） |
| **扩展预留** | 预留状态枚举、字段长度，支持后续功能扩展 |

## 2. 核心实体清单

| 实体 | 说明 | 来源 | 对应表 |
|------|------|------|--------|
| **Seat** | 座位信息 | PRD-Lite §11 | Seat |
| **Reservation** | 预约记录 | PRD-Lite §11 | Reservation |
| **AdminUser** | 管理员账号 | PRD-Lite §11 | AdminUser |

> **说明**：前序文档明确只建 3 张核心表，不引入 User、SeatArea、TimeSlot、ReservationLog 等额外实体。体验账号通过 Session 模拟，不建用户表；时段固定为 5 个值，通过字符串存储；不记录操作日志。

## 3. 关系说明

### 3.1 ER 关系图

```
┌─────────────────┐         ┌─────────────────────┐
│      Seat       │         │    Reservation      │
├─────────────────┤         ├─────────────────────┤
│ Id (PK)         │◄────────│ SeatId (FK)         │
│ Name (UQ)       │    1:N  │ UserName            │
│ Location        │         │ ReserveDate         │
│ HasPower        │         │ TimeSlot            │
│ Status          │         │ Status              │
│ CreatedAt       │         │ CreatedAt           │
└─────────────────┘         └─────────────────────┘
        │                           │
        │                           │
        │  Seat 1 ──── N Reservation │
        │                           │
        └───────────────────────────┘

┌─────────────────┐
│   AdminUser     │
├─────────────────┤
│ Id (PK)         │
│ Username (UQ)   │
│ Password        │
└─────────────────┘
      独立表
```

### 3.2 关系说明

| 关系 | 类型 | 说明 |
|------|------|------|
| Seat → Reservation | 一对多 | 一个座位可有多条预约记录 |
| Reservation → Seat | 多对一 | 每条预约记录关联一个座位 |
| AdminUser | 独立 | 管理员表与预约表无直接关联 |

### 3.3 外键约束

| 表 | 外键字段 | 引用表 | 引用字段 | 级联规则 |
|----|----------|--------|----------|----------|
| Reservation | SeatId | Seat | Id | NO ACTION |

> **设计决策**：删除有预约记录的座位时，应在应用层校验并阻止，而非依赖数据库级联删除。

## 4. 表结构设计

### 4.1 Seat 表（座位表）

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| Id | int | PK, Identity(1,1) | - | 座位 ID |
| Name | nvarchar(50) | NOT NULL, UNIQUE | - | 座位编号（如 "A-01"） |
| Location | nvarchar(100) | NOT NULL | - | 位置描述（如 "一楼东侧"） |
| HasPower | bit | NOT NULL | 0 | 是否有电源（0=无, 1=有） |
| Status | int | NOT NULL | 0 | 状态（0=可用, 1=维护中） |
| CreatedAt | datetime2 | NOT NULL | GETUTCDATE() | 创建时间 |

**索引**：
| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| PK_Seat | Id | 聚簇唯一 | 主键 |
| IX_Seat_Name | Name | 唯一 | 座位编号不重复 |
| IX_Seat_Status | Status | 普通 | 按状态筛选可用座位 |

**种子数据**：20 条记录（A区5个 + B区5个 + C区5个 + D区5个）

---

### 4.2 Reservation 表（预约记录表）

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| Id | int | PK, Identity(1,1) | - | 预约 ID |
| SeatId | int | FK → Seat.Id, NOT NULL | - | 座位 ID |
| UserName | nvarchar(50) | NOT NULL | - | 预约人姓名（体验账号） |
| ReserveDate | datetime2 | NOT NULL | - | 预约日期 |
| TimeSlot | nvarchar(20) | NOT NULL | - | 时段（固定 5 个值之一） |
| Status | int | NOT NULL | 0 | 状态（0=已预约, 1=已签到, 2=已取消, 3=已过期） |
| CreatedAt | datetime2 | NOT NULL | GETUTCDATE() | 创建时间 |

**索引**：
| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| PK_Reservation | Id | 聚簇唯一 | 主键 |
| IX_Reservation_SeatId_Date_TimeSlot | SeatId + ReserveDate + TimeSlot | 唯一 | 冲突校验（同座位+同时段不可重复） |
| IX_Reservation_UserName | UserName | 普通 | 查询"我的预约" |
| IX_Reservation_Status | Status | 普通 | 按状态筛选 |
| IX_Reservation_ReserveDate | ReserveDate | 普通 | 按日期筛选 |

**外键约束**：
| 约束名 | 字段 | 引用 | 级联规则 |
|--------|------|------|----------|
| FK_Reservation_Seat | SeatId | Seat.Id | NO ACTION |

---

### 4.3 AdminUser 表（管理员表）

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| Id | int | PK, Identity(1,1) | - | 管理员 ID |
| Username | nvarchar(50) | NOT NULL, UNIQUE | - | 登录账号 |
| Password | nvarchar(100) | NOT NULL | - | 密码哈希（PasswordHasher） |
| CreatedAt | datetime2 | NOT NULL | GETUTCDATE() | 创建时间 |

**索引**：
| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| PK_AdminUser | Id | 聚簇唯一 | 主键 |
| IX_AdminUser_Username | Username | 唯一 | 账号不重复 |

**种子数据**：1 条记录（admin / 123456）

## 5. 关键字段说明

### 5.1 座位编号（Seat.Name）

| 属性 | 说明 |
|------|------|
| 格式 | 字母-数字（如 "A-01"、"C-04"） |
| 约束 | UNIQUE，不可重复 |
| 用途 | 用户端展示、管理员管理、预约记录引用 |

### 5.2 预约人姓名（Reservation.UserName）

| 属性 | 说明 |
|------|------|
| 格式 | 中文姓名（如 "张三"、"李四"） |
| 来源 | Session 中的 CurrentUserName |
| 约束 | 非空，不关联用户表 |
| 说明 | 体验账号模拟，无密码验证 |

### 5.3 时段（Reservation.TimeSlot）

| 时段值 | 显示名称 | 时间范围 |
|--------|----------|----------|
| Morning1 | 上午第一节 | 08:00-10:00 |
| Morning2 | 上午第二节 | 10:00-12:00 |
| Afternoon1 | 下午第一节 | 14:00-16:00 |
| Afternoon2 | 下午第二节 | 16:00-18:00 |
| Evening | 晚上 | 19:00-21:00 |

> **说明**：时段固定为 5 个值，通过字符串存储，不单独建表。

### 5.4 预约日期（Reservation.ReserveDate）

| 属性 | 说明 |
|------|------|
| 类型 | datetime2 |
| 比较方式 | 使用 `.Date` 属性忽略时间部分 |
| 默认值 | 今天 |
| 可选范围 | 今天及之后 |

## 6. 状态字段设计

### 6.1 座位状态（Seat.Status）

| 状态值 | 枚举名 | 显示名称 | 说明 |
|--------|--------|----------|------|
| 0 | Available | 可用 | 正常可预约 |
| 1 | UnderMaintenance | 维护中 | 不可预约，详情页提示 |

**状态流转**：
```
0 (可用) ←──管理员编辑──→ 1 (维护中)
```

**业务规则**：
- 管理员可在座位管理页编辑状态
- 维护中的座位：用户端列表仍显示，但详情页提示"维护中，暂不可预约"
- 维护中的座位不能被预约

---

### 6.2 预约状态（Reservation.Status）

| 状态值 | 枚举名 | 显示名称 | 说明 |
|--------|--------|----------|------|
| 0 | Reserved | 已预约 | 有效预约 |
| 1 | CheckedIn | 已签到 | 本期不使用 |
| 2 | Cancelled | 已取消 | 用户主动取消 |
| 3 | Expired | 已过期 | 管理员手动标记（本期可选） |

**状态流转**：
```
0 (已预约) ──用户取消──→ 2 (已取消)
0 (已预约) ──过期──→ 3 (已过期)  [可选]
```

**业务规则**：
- 0→2：用户在"我的预约页"点击取消
- 0→3：管理员在预约管理页手动标记（本期可选）
- 状态不可逆：已取消的预约不能再恢复
- 冲突校验只检查 Status=0 的记录

## 7. 查询与筛选字段设计

### 7.1 查询场景矩阵

| 查询场景 | 筛选条件 | 排序方式 | 对应页面 |
|----------|----------|----------|----------|
| 座位列表 | 查询所有座位 | Id ASC | 用户端座位列表 |
| 座位详情 | Id = @id | - | 用户端座位详情 |
| 时段状态 | SeatId = @id AND ReserveDate = @today | TimeSlot ASC | 用户端座位详情 |
| 冲突校验 | SeatId + ReserveDate + TimeSlot + Status = 0 | - | 预约提交 |
| 我的预约 | UserName = @name | ReserveDate DESC | 用户端我的预约 |
| 预约管理 | ReserveDate = @date, Status = @status | ReserveDate DESC | 管理端预约管理 |
| 热门座位 | Status ≠ 2 | COUNT(*) DESC | 管理端统计 |
| 时段分布 | Status ≠ 2 | COUNT(*) DESC | 管理端统计 |

### 7.2 索引覆盖设计

| 查询场景 | 覆盖索引 | 说明 |
|----------|----------|------|
| 冲突校验 | IX_Reservation_SeatId_Date_TimeSlot | 复合索引，覆盖主查询条件 |
| 我的预约 | IX_Reservation_UserName + IX_Reservation_Status | 双索引，支持用户名+状态筛选 |
| 预约管理 | IX_Reservation_ReserveDate + IX_Reservation_Status | 双索引，支持日期+状态筛选 |
| 时段状态 | IX_Reservation_SeatId_Date_TimeSlot | 复合索引，覆盖座位+日期查询 |
| 可用座位 | IX_Seat_Status | 单索引，覆盖状态筛选 |

## 8. 统计字段与统计口径

### 8.1 预约状态统计

| 统计项 | 计算公式 | 说明 |
|--------|----------|------|
| 总预约数 | `COUNT(*) FROM Reservation` | 所有记录，包含所有状态 |
| 今日预约数 | `COUNT(*) FROM Reservation WHERE ReserveDate = 今天 AND Status ≠ 2` | 排除已取消的记录 |
| 有效预约数 | `COUNT(*) FROM Reservation WHERE Status = 0` | 仅"已预约"状态 |
| 已取消数 | `COUNT(*) FROM Reservation WHERE Status = 2` | 仅"已取消"状态 |
| 已过期数 | `COUNT(*) FROM Reservation WHERE Status = 3` | 仅"已过期"状态 |

### 8.2 热门座位统计

| 统计项 | 计算公式 | 说明 |
|--------|----------|------|
| 热门座位 TOP5 | 按 `SeatId` 分组，`COUNT(*)` 降序排列 | 相同次数按 `SeatId` 升序 |
| 排除条件 | 排除 `Status = 2` 的记录 | 已取消的预约不计入统计 |

### 8.3 时段分布统计

| 统计项 | 计算公式 | 说明 |
|--------|----------|------|
| 时段分布 | 按 `TimeSlot` 分组，`COUNT(*)` 计数 | 仅统计 `Status ≠ 2` 的记录 |
| 占比计算 | `该时段预约数 / 总预约数 × 100%` | 四舍五入保留 1 位小数 |
| 100% 校验 | 各时段占比相加应为 100%（±0.1% 误差可接受） | 由于四舍五入可能存在微小误差 |

### 8.4 "今天"判断标准

| 项目 | 口径 |
|------|------|
| 判断方式 | 基于 `ReserveDate` 字段（日期部分），忽略时间部分 |
| 代码实现 | `ReserveDate.Date == DateTime.Today` |
| 实时性 | 统计数据实时计算，无缓存，每次访问重新查询 |

## 9. 数据一致性与约束说明

### 9.1 唯一约束

| 表 | 字段组合 | 约束 | 业务含义 |
|----|----------|------|----------|
| Seat | Name | UNIQUE | 座位编号不可重复 |
| AdminUser | Username | UNIQUE | 管理员账号不可重复 |
| Reservation | SeatId + ReserveDate + TimeSlot | COMPOSITE UNIQUE | 同一座位同一日期同一时段只能预约一次 |

### 9.2 外键约束

| 表 | 字段 | 引用表 | 引用字段 | 级联规则 | 说明 |
|----|------|--------|----------|----------|------|
| Reservation | SeatId | Seat | Id | NO ACTION | 删除座位前需校验无关联预约 |

### 9.3 默认值

| 表 | 字段 | 默认值 | 说明 |
|----|------|--------|------|
| Seat | HasPower | 0 | 默认无电源 |
| Seat | Status | 0 | 默认可用 |
| Reservation | Status | 0 | 默认已预约 |
| Seat | CreatedAt | GETUTCDATE() | 创建时间 |
| Reservation | CreatedAt | GETUTCDATE() | 创建时间 |
| AdminUser | CreatedAt | GETUTCDATE() | 创建时间 |

### 9.4 应用层校验规则

| 校验场景 | 校验规则 | 错误提示 |
|----------|----------|----------|
| 新增座位 | Name 不可重复 | "座位编号已存在" |
| 编辑座位 | Name 不可重复（排除自身） | "座位编号已存在" |
| 删除座位 | 无关联 Reservation 记录 | "该座位有预约记录，无法删除" |
| 提交预约 | SeatId + ReserveDate + TimeSlot 不重复（Status=0） | "该时段已被预约" |
| 取消预约 | 只能取消自己的预约 | "无法取消他人预约" |
| 取消预约 | 只能取消 Status=0 的预约 | "该预约已取消" |
