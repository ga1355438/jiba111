# 项目立项单

## 1. 项目基本信息

| 项目 | 内容 |
|------|------|
| 项目名称 | 图书馆座位预约系统 |
| 项目类型 | 课程实战项目（模拟独立负责） |
| 技术栈 | ASP.NET Core MVC (.NET 8) + Razor + SQL Server LocalDB + EF Core + Bootstrap |
| 代码仓库 | GitHub 或 Gitee（项目结束时提交链接） |
| 文档版本 | v1.0 |

## 2. 项目背景

高校图书馆高峰期座位紧张，学生到馆后经常找不到座位，或者出现占座不用的情况。本项目模拟开发一个轻量级座位预约系统，让学生可以提前在线预约座位，到馆后签到使用，管理员可以查看预约情况并进行管理。

本项目作为课程第三阶段实战任务，模拟"入职后独立负责第一个小项目"的真实工作流程，要求从立项、需求、设计、开发到验收交付全流程完成。

## 3. 目标用户

| 角色 | 说明 |
|------|------|
| 普通用户（学生） | 需要预约图书馆座位的学生，使用体验账号登录即可操作 |
| 管理员 | 负责管理座位信息和查看预约记录，使用 Session/Cookie 登录 |

## 4. 项目目标

- 能让用户浏览可用座位并完成预约
- 能让用户查看和取消自己的预约
- 能让管理员管理座位（增删改）和查看所有预约
- 能提供基本的预约统计视图


## 5. 本期必须完成内容

### 用户端

| 序号 | 功能 | 说明 |
|------|------|------|
| 1 | 用户首页 | 展示欢迎页、导航入口 |
| 2 | 座位列表 | 分页展示所有座位及当前状态（可用/已预约） |
| 3 | 座位详情 | 展示单个座位信息，含所选日期各时段预约状态 |
| 4 | 预约提交 | 选择时段后提交预约（默认体验账号） |
| 5 | 我的预约 | 查看当前用户的所有预约记录 |
| 6 | 取消预约 | 用户可取消未开始的预约 |

### 管理端

| 序号 | 功能 | 说明 |
|------|------|------|
| 1 | 管理员登录 | 轻量 Session/Cookie 登录，预设账号密码 |
| 2 | 预约管理 | 查看所有预约，支持按条件筛选 |
| 3 | 座位管理 | 增加、编辑、删除座位信息 |
| 4 | 统计页 | 展示预约总数、各座位使用率等基本统计 |

- 扫码签到

## 8. 关键业务主流程

```
用户首页 → 座位列表 → 座位详情 → 选择时段 → 预约提交 → 我的预约 → 取消预约（可选）
                                     
管理员登录 → 座位管理 / 预约管理 / 统计页
```

核心流程（用户预约）：
1. 用户进入座位列表，浏览所有座位
2. 点击某个座位，进入详情页
3. 选择预约日期和时段
4. 点击提交预约
5. 系统校验时段冲突，写入数据库
6. 跳转到"我的预约"确认结果

核心流程（管理员管理）：
1. 管理员输入预设账号密码登录
2. 可在座位管理页增删改座位
3. 可在预约管理页查看所有用户的预约记录
4. 可在统计页查看总预约数和座位使用概况

## 9. 成功标准

完成以下动作即视为项目成功：

- [ ] 能启动项目，浏览器正常打开用户首页
- [ ] 能在座位列表看到预置的座位数据
- [ ] 能选择一个座位并成功提交预约
- [ ] 能在"我的预约"中看到刚提交的预约记录
- [ ] 能成功取消一条未开始的预约
- [ ] 管理员能使用预设账号登录
- [ ] 管理员能新增、编辑、删除一个座位
- [ ] 管理员能查看所有预约记录
- [ ] 管理员统计页能显示预约总数
- [ ] 同一座位同一时段无法重复预约（提交时提示冲突）

## 10. 风险与假设

| 风险 | 影响 | 应对 |
|------|------|------|
| 时间不够 | 功能做不完，验收不过 | 优先保证用户端预约流程，管理端统计页可简化 |
| 页面失控 | 页面太多写不过来 | 严格控制在 10 页以内，不加新页面 |
| 数据库过重 | 表结构设计拖延进度 | 只建 Seat、Reservation、AdminUser 三张表，够用就停 |
| AI 辅助代码不稳定 | 生成的代码跑不起来 | 每写完一个功能立刻运行验证，不堆积 |
| Razor 视图语法不熟 | 页面渲染出错 | 遇到问题先查官方文档，不要死磕 |
| EF Core 迁移出错 | 数据库状态混乱 | 每次迁移前备份，出错就重建数据库 |
