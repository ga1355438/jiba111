## 2. 本地部署

### 2.1 获取代码

```bash
# 克隆仓库（如果从 Git 获取）
git clone <仓库地址>
cd LibrarySeatSystem

# 或直接进入项目目录
cd D:\game\111\LibrarySeatSystem
```

### 2.2 还原依赖

```bash
dotnet restore
```

### 2.3 数据库初始化

数据库会在首次运行时自动创建并填充种子数据。

如需手动初始化：

```bash
dotnet ef database update
```

### 2.4 编译项目

```bash
dotnet build
```

### 2.5 运行项目

```bash
# 方式一：指定端口
dotnet run --urls "http://localhost:5000"

# 方式二：使用多个端口
dotnet run --urls "http://localhost:5000;https://localhost:5001"

# 方式三：使用 0.0.0.0（允许外部访问）
dotnet run --urls "http://0.0.0.0:5000"
```

### 2.6 访问系统

| 端口 | 地址 | 说明 |
|------|------|------|
| 用户端 | http://localhost:5000 | 学生用户使用 |
| 管理端 | http://localhost:5000/Admin/Login | 管理员登录 |

## 3. 生产环境部署

### 3.1 发布项目

```bash
dotnet publish -c Release -o ./publish
```

### 3.2 部署到 IIS（Windows）

1. 安装 .NET Hosting Bundle
2. 创建 IIS 网站
3. 指向 publish 目录
4. 设置应用程序池为"无托管代码"

### 3.3 部署到 Linux

```bash
# 安装 .NET Runtime
sudo apt-get install -y dotnet-runtime-6.0

# 复制 publish 目录到服务器
scp -r ./publish user@server:/var/www/libraryseat

# 在服务器上运行
cd /var/www/libraryseat
dotnet LibrarySeatSystem.dll --urls "http://0.0.0.0:5000"
```

### 3.4 使用 Docker（可选）

创建 Dockerfile：

```dockerfile
FROM mcr.microsoft.com/dotnet/aspnet:6.0
WORKDIR /app
COPY ./publish .
ENTRYPOINT ["dotnet", "LibrarySeatSystem.dll"]
```

构建和运行：

```bash
docker build -t libraryseat .
docker run -d -p 5000:80 --name libraryseat libraryseat
```

## 4. 配置说明

### 4.1 应用配置（appsettings.json）

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=(localdb)\\mssqllocaldb;Database=LibrarySeatSystem;Trusted_Connection=True;"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information"
    }
  }
}
```

### 4.2 Session 配置

- Session 超时时间：30 分钟
- 存储方式：内存（InMemory）

### 4.3 数据库连接

默认使用 SQL Server LocalDB，连接字符串在 `appsettings.json` 中配置。

如需使用其他数据库，修改连接字符串并安装对应的 EF Core 提供程序。

## 6. 常见问题

### 6.1 端口被占用

```bash
# 查找占用端口的进程
netstat -ano | findstr :5000

# 终止进程
taskkill /F /PID <进程ID>
```

### 6.2 数据库连接失败

1. 检查 SQL Server LocalDB 是否运行
2. 检查连接字符串是否正确
3. 尝试使用 `sqllocaldb start` 启动 LocalDB

### 6.3 编译失败

```bash
# 清理并重新编译
dotnet clean
dotnet build
```

### 6.4 样式不生效

1. 清除浏览器缓存（Ctrl+F5）
2. 检查 `wwwroot/css/site.css` 是否正确加载
