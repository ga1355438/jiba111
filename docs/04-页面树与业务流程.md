## 4. 用户主链路

### 主链路 1：学生预约座位（核心闭环）

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 用户首页 │───→│座位列表页│───→│座位详情页│───→│预约提交页│───→│我的预约页│
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
   点击            点击            点击            提交成功         完成
  "查看座位"     "查看详情"     "预约此座位"     跳转             预约成功
```

**流程说明**：
1. 用户访问**用户首页**，点击"查看座位"
2. 跳转**座位列表页**，浏览座位信息，点击"查看详情"
3. 跳转**座位详情页**，查看时段状态，点击"预约此座位"
4. 跳转**预约提交页**，选择日期和时段，点击"提交预约"
5. 提交成功后跳转**我的预约页**，显示预约记录

### 主链路 2：学生取消预约

```
┌─────────┐    ┌─────────┐
│我的预约页│───→│我的预约页│
└─────────┘    └─────────┘
   点击            刷新
  "取消"         状态变更
```

**流程说明**：
1. 用户在**我的预约页**点击"取消"按钮
2. 弹出确认框"确定取消此预约吗？"
3. 确认后 POST `/Reservation/Cancel/{id}`
4. 页面刷新，状态变为"已取消"

### 主链路 3：学生切换体验账号

```
┌─────────┐    ┌─────────┐
│任意用户端│───→│任意用户端│
│  页面    │    │  页面    │
└─────────┘    └─────────┘
   选择            刷新
  账号            数据联动
```

**流程说明**：
1. 用户在导航栏下拉框选择账号（张三/李四/王五）
2. POST `/Reservation/SwitchUser`
3. Session 更新，当前页面刷新，数据按新账号筛选

## 5. 管理主链路

### 主链路 1：管理员登录

```
┌─────────┐    ┌─────────┐
│管理员登录│───→│座位管理页│
└─────────┘    └─────────┘
   登录成功       进入后台
```

**流程说明**：
1. 管理员访问**管理员登录页**（未登录自动跳转）
2. 输入用户名/密码（admin/123456）
3. 登录成功后跳转**座位管理页**
4. 登录失败显示"用户名或密码错误"

### 主链路 2：管理员管理座位

```
┌─────────┐    ┌─────────┐
│座位管理页│───→│座位管理页│
└─────────┘    └─────────┘
   操作            刷新
 增/删/改        列表更新
```

**流程说明**：
1. 管理员在**座位管理页**查看座位列表
2. 新增：点击"新增座位" → 填写表单 → 提交 → 列表刷新
3. 编辑：点击"编辑" → 修改信息 → 提交 → 列表刷新
4. 删除：点击"删除" → 确认 → POST 删除 → 列表刷新

### 主链路 3：管理员查看预约与统计

```
┌─────────┐    ┌─────────┐
│预约管理页│    │ 统计页  │
└─────────┘    └─────────┘
   筛选           查看
  查询           数据
```

**流程说明**：
1. 管理员在**预约管理页**按日期/状态筛选预约记录
2. 管理员在**统计页**查看总预约数、今日预约数、热门座位、时段分布

## 6. 状态流转

### 6.1 座位状态（Seat.Status）

| 状态值 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 可用 | 正常可预约 |
| 1 | 维护中 | 不可预约，详情页提示 |

**状态流转**：
```
0 (可用) ←──编辑──→ 1 (维护中)
```

- 管理员可在座位管理页编辑状态
- 维护中的座位：用户端列表仍显示，但详情页提示"维护中，暂不可预约"

### 6.2 预约状态（Reservation.Status）

| 状态值 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 已预约 | 有效预约 |
| 1 | 已签到 | 本期不使用 |
| 2 | 已取消 | 用户主动取消 |
| 3 | 已过期 | 本期可选 |

**状态流转**：
```
0 (已预约) ──取消──→ 2 (已取消)
0 (已预约) ──过期──→ 3 (已过期)  [可选]
```

- 0→2：用户在"我的预约页"点击取消
- 0→3：管理员在预约管理页手动标记（本期可选）
- 状态不可逆：已取消的预约不能恢复

### 6.3 管理员会话（Session）

```
未登录 ──登录成功──→ 已登录（AdminUsername 写入 Session）
                       │
                       ├──退出登录──→ 未登录（Session 清除）
                       └──超时30分钟──→ 未登录（自动失效）
```

## 7. 页面与流程关系说明

### 7.1 页面职责定义

| 页面名称 | 核心职责 | 主要交互 |
|----------|----------|----------|
| 用户首页 | 系统入口，提供快速导航 | 2 个入口按钮 |
| 座位列表页 | 展示所有座位，支持分页 | 表格浏览、分页、查看详情 |
| 座位详情页 | 展示单个座位信息和时段状态 | 时段状态展示、预约入口 |
| 预约提交页 | 提交预约请求 | 日期选择、时段选择、表单提交 |
| 我的预约页 | 查看当前用户预约记录 | 预约列表、取消操作、账号切换 |
| 管理员登录页 | 管理员认证入口 | 用户名密码表单 |
| 座位管理页 | 管理座位资源 | 列表展示、新增/编辑/删除 |
| 预约管理页 | 查看所有预约记录 | 筛选查询、列表展示 |
| 统计页 | 展示预约统计数据 | 数据可视化展示 |

### 7.2 页面跳转矩阵

| 源页面 | 目标页面 | 触发条件 |
|--------|----------|----------|
| 用户首页 | 座位列表页 | 点击"查看座位" |
| 用户首页 | 我的预约页 | 点击"我的预约" |
| 座位列表页 | 座位详情页 | 点击"查看详情" |
| 座位详情页 | 预约提交页 | 点击"预约此座位" |
| 座位详情页 | 座位列表页 | 点击"返回列表" |
| 预约提交页 | 我的预约页 | 提交成功 |
| 预约提交页 | 座位详情页 | 点击"返回" |
| 管理员登录页 | 座位管理页 | 登录成功 |
| 座位管理页 | 座位新增表单页 | 点击"新增座位" |
| 座位管理页 | 座位编辑表单页 | 点击"编辑" |
| 座位新增/编辑表单页 | 座位管理页 | 提交成功/取消 |

### 7.3 操作与页面关系

| 操作名称 | 操作方式 | 所在页面 | 目标 |
|----------|----------|----------|------|
| 取消预约 | POST | 我的预约页 | 刷新当前页 |
| 切换体验账号 | POST | 导航栏 | 刷新当前页 |
| 管理员登出 | GET | 导航栏 | 跳转登录页 |
| 座位新增 | GET/POST | 座位管理页 | 跳转表单页 |
| 座位编辑 | GET/POST | 座位管理页 | 跳转表单页 |
| 座位删除 | POST | 座位管理页 | 刷新当前页 |
