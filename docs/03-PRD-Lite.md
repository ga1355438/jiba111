## 1. 项目概述

高校图书馆高峰期座位紧张，学生到馆后经常找不到座位，或出现占座不用的情况。本项目开发一个轻量级座位预约系统，让学生可以提前在线预约座位，管理员可以管理座位资源和查看预约情况。

系统分为用户端和管理端。用户端面向学生，提供座位浏览、预约、查看和取消预约等功能；管理端面向图书馆管理员，提供座位管理、预约管理和统计查看等功能。

技术栈：ASP.NET Core MVC (.NET 8) + Razor + SQL Server LocalDB + EF Core + Bootstrap

---

## 2. 项目目标

| 目标 | 衡量方式 |
|------|----------|
| 用户能浏览可用座位并完成预约 | 从座位列表到预约提交，全流程可操作 |
| 用户能查看和取消自己的预约 | "我的预约"页可查看记录，可取消未开始的预约 |
| 管理员能管理座位和查看预约 | 座位管理页可增删改，预约管理页可筛选查看 |
| 系统能提供基本的预约统计 | 统计页展示总预约数、热门座位、时段分布 |

## 4. 使用场景

### 场景 A — 学生预约座位（主线）

> 小明下午想自习，打开系统，浏览座位列表看到三楼 C-04 有电源且可约，进入详情看到 14:00-16:00 时段空闲，点击"预约此座位"，选择日期和时段后提交。系统校验通过后跳转到"我的预约"，小明看到这条记录状态为"已预约"。临时有事去不了，小明在"我的预约"点击"取消"，确认后状态变为"已取消"。

异常：若所选时段已被预约，系统提示"该时段已被预约，请选择其他时段"，不写入数据。

### 场景 B — 管理员管理座位（主线）

> 管理员登录后台，在座位管理页看到现有 20 个座位。新到了一批座位，管理员点击"新增座位"，填写编号 A-06、位置一楼东侧、有电源，提交后列表新增一条。发现 D-05 坏了，点击"编辑"将其状态改为"维护中"。想删除一个没有预约记录的座位，点击"删除"确认后移除。

异常：新增编号重复时提示"座位编号已存在"；删除有预约记录的座位时阻止并提示。

### 场景 C — 管理员查看统计（支线）

> 管理员打开统计页，看到总预约数 35、今日预约数 8、有效预约数 12。热门座位 TOP5 显示 C-04 被预约了 12 次排第一。时段分布显示 14:00-16:00 占比最高。

异常：无预约数据时，各统计项显示"0"，不报错。

## 7. 页面清单

共计 **9 个视图页面**（取消预约为 POST 操作，不独立成页）

> **说明**：与 02-需求分析与MVP确认.md 的差异：MVP确认文档中取消预约（`/Reservation/Cancel/{id}`）计为独立页面，共10页。本文档将其归为POST操作，不计入页面数。两种口径均合理，本文档采用9页口径以更贴近实际开发实现。

| 序号 | 端 | 页面名称 | 路由 | HTTP |
|------|-----|----------|------|------|
| 1 | 用户端 | 首页 | `/Home/Index` | GET |
| 2 | 用户端 | 座位列表 | `/Seat/List` | GET |
| 3 | 用户端 | 座位详情 | `/Seat/Detail/{id}` | GET |
| 4 | 用户端 | 预约提交 | `/Reservation/Create/{seatId}` | GET/POST |
| 5 | 用户端 | 我的预约 | `/Reservation/My` | GET |
| 6 | 管理端 | 登录 | `/Admin/Login` | GET/POST |
| 7 | 管理端 | 座位管理 | `/Admin/Seat/Index` | GET |
| 8 | 管理端 | 预约管理 | `/Admin/Reservation/Index` | GET |
| 9 | 管理端 | 统计页 | `/Admin/Statistics/Index` | GET |

辅助路由（POST 操作，不独立成页）：

> 座位新增和编辑的表单页（`/Admin/SeatCreate`、`/Admin/SeatEdit/{id}`）视为座位管理页的子页面，不单独计入 9 页口径。

| 操作 | 路由 | HTTP |
|------|------|------|
| 取消预约 | `/Reservation/Cancel/{id}` | POST |
| 切换体验账号 | `/Reservation/SwitchUser` | POST |
| 管理员登出 | `/Admin/Logout` | GET |
| 座位新增 | `/Admin/SeatCreate` | GET/POST |
| 座位编辑 | `/Admin/SeatEdit/{id}` | GET/POST |
| 座位删除 | `/Admin/SeatDelete/{id}` | POST |

## 8. 核心业务流程

### 8.1 用户预约流程

```
首页 → 座位列表 → 座位详情 → 选择时段 → 预约提交 → 我的预约
                          ↓ (冲突)
                      提示错误，返回详情
```

### 8.2 用户取消流程

```
我的预约 → 点击取消 → 确认弹窗 → POST 取消 → 刷新我的预约
```

### 8.3 管理员登录流程

```
登录页 → 输入账号密码 → 校验 → 成功: 跳转座位管理 / 失败: 显示错误
```

### 8.4 管理员座位管理流程

```
座位管理列表 → 新增/编辑/删除 → 表单提交 → 校验 → 成功: 刷新列表 / 失败: 显示错误
```

### 8.5 管理员预约查看流程

```
预约管理列表 → 选择筛选条件 → 查询 → 刷新列表
```

| 统计项 | 计算公式 | 说明 |
|--------|----------|------|
| 总预约数 | `COUNT(*) FROM Reservation` | 所有记录，包含所有状态 |
| 今日预约数 | `COUNT(*) FROM Reservation WHERE ReserveDate = 今天 AND Status != 2` | 排除已取消的记录 |
| 有效预约数 | `COUNT(*) FROM Reservation WHERE Status = 0` | 仅"已预约"状态 |
| 已取消数 | `COUNT(*) FROM Reservation WHERE Status = 2` | 仅"已取消"状态 |
| 已过期数 | `COUNT(*) FROM Reservation WHERE Status = 3` | 仅"已过期"状态 |

### 热门座位统计口径

| 统计项 | 计算公式 | 说明 |
|--------|----------|------|
| 热门座位 TOP5 | 按 `SeatId` 分组，`COUNT(*)` 降序排列 | 相同次数按 `SeatId` 升序 |
| 排除条件 | 排除 `Status = 2` 的记录 | 已取消的预约不计入统计 |

### 时段分布统计口径

| 统计项 | 计算公式 | 说明 |
|--------|----------|------|
| 时段分布 | 按 `TimeSlot` 分组，`COUNT(*)` 计数 | 仅统计 `Status != 2` 的记录 |
| 占比计算 | `该时段预约数 / 总预约数 × 100%` | 四舍五入保留 1 位小数 |
| 100% 校验 | 各时段占比相加应为 100%（±0.1% 误差可接受） | 由于四舍五入可能存在微小误差 |

### 其他口径

| 项目 | 口径 |
|------|------|
| "今天" 判断标准 | 基于 `ReserveDate` 字段（日期部分），忽略时间部分 |
| 日期比较 | `ReserveDate.Date == DateTime.Today` |
| 实时性 | 统计数据实时计算，无缓存，每次访问重新查询 |

## 13. 状态流转说明

### 座位状态（Seat.Status）

```
0 (可用) ←→ 1 (维护中)
```

- 管理员可将可用座位改为维护中（编辑操作）
- 管理员可将维护中座位改回可用（编辑操作）
- 维护中的座位不在用户端可预约列表中突出显示，但仍在列表中（状态标记为维护中，详情页提示不可预约）

### 预约状态（Reservation.Status）

```
0 (已预约) → 2 (已取消)
0 (已预约) → 3 (已过期)
```

- 0 → 2：用户主动取消（我的预约页点击取消）
- 0 → 3：管理员手动标记（预约管理页，本期可选）
- 1（已签到）：本期不做
- 状态不可逆：已取消的预约不能再恢复

### 管理员会话（Session）

```
未登录 → 登录成功 → AdminUsername 写入 Session
→ 退出登录 → Session 清除
→ Session 超时（30 分钟） → 自动失效
```

## 16. 风险与假设

| 风险 | 影响 | 应对 |
|------|------|------|
| 时间不够 | 功能做不完 | 优先保证用户端预约流程，管理端统计页可简化 |
| 页面失控 | 页面太多写不过来 | 严格控制在 9 页视图页面以内，不加新页面（取消预约等POST操作不计入） |
| 数据库过重 | 表结构设计拖延进度 | 只建 3 张表，够用就停 |
| AI 辅助代码不稳定 | 生成的代码跑不起来 | 每写完一个功能立刻运行验证 |
| Razor 视图语法不熟 | 页面渲染出错 | 遇到问题先查官方文档 |
| EF Core 迁移出错 | 数据库状态混乱 | 出错就重建数据库 |
| 需求变更 | 开发中想加功能 | 严格按本文档范围执行，不加新功能 |

### 假设

1. SQL Server LocalDB 已安装并可用
2. .NET 8 SDK 已安装（当前环境使用 .NET 6，API 兼容）
3. 开发者有基本的 C# 和 ASP.NET Core 基础
4. 课堂项目不需要考虑并发、性能、安全等生产级问题
