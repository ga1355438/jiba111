### 2.7 异常与边界情况

根据 08-数据库设计.md §9.4：

| 异常场景 | 校验规则 | 错误提示 | 来源 |
|----------|----------|----------|------|
| 座位不存在 | Id 无效 | NotFound() | 通用 |
| 座位维护中 | Status = 1 | "维护中，暂不可预约" | 04-§6.1 |
| 时段已预约 | SeatId + ReserveDate + TimeSlot + Status=0 存在 | "该时段已被预约" | 08-§9.4 |
| 唯一索引冲突 | 并发场景 | "该时段已被预约" | 08-§9.1 |
| 数据库异常 | 任何写入操作 | "提交失败，请稍后重试" | 07-§8 |

**说明**：冲突校验（SELECT COUNT）和写入（INSERT）之间存在时间窗口，由数据库唯一索引兜底。INSERT 冲突时返回 ViewBag.Error = "该时段已被预约"。

根据 08-数据库设计.md §7.1、§8：

| 步骤 | 操作 | SQL/EF Core | 索引覆盖 |
|------|------|-------------|----------|
| 1 | 查询所有预约 | `SELECT * FROM Reservation LEFT JOIN Seat ON Reservation.SeatId = Seat.Id` | - |
| 1 | 从内存计算总预约数 | `reservations.Count` | - |
| 1 | 从内存计算今日预约数 | `reservations.Count(r => r.ReserveDate.Date == today && r.Status != 2)` | - |
| 1 | 从内存计算有效预约数 | `reservations.Count(r => r.Status == 0)` | - |
| 1 | 从内存计算热门座位 TOP5 | `reservations.GroupBy(r => r.SeatId).Select(g => new { SeatId = g.Key, Count = g.Count() }).OrderByDescending(x => x.Count).Take(5)` | - |
| 1 | 从内存计算时段分布 | `reservations.GroupBy(r => r.TimeSlot).Select(g => new { Slot = g.Key, Count = g.Count() }).OrderByDescending(x => x.Count)` | - |
