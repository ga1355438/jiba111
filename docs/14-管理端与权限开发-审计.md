### B-4. 座位新增缺少编号唯一性校验（P2-建议优化）

docs/08 §9.4 明确要求：

> 新增座位：Name 不可重复 → "座位编号已存在"

代码实现（AdminController.cs:67-72）：

```csharp
[HttpPost]
public async Task<IActionResult> SeatCreate(Seat seat)
{
    if (!IsLoggedIn()) return RedirectToAction(nameof(Login));
    seat.CreatedAt = DateTime.UtcNow;
    await _seatRepository.AddAsync(seat);
    return RedirectToAction(nameof(SeatIndex));
}
```

SeatRepository 存在 `ExistsByNameAsync()` 方法，但 AdminController 未调用。若用户输入重复编号，EF Core 唯一索引会抛异常，而非返回友好提示。

**影响**：用户体验差（看到 500 错误页面而非"座位编号已存在"提示）。数据库层面不会写入重复数据（唯一索引兜底）。

**建议修复**：在 SeatCreate POST 中增加 `ExistsByNameAsync` 校验。

---

### B-5. 座位删除未在应用层校验关联预约（P2-建议优化）

docs/08 §9.4 明确要求：

> 删除座位：无关联 Reservation 记录 → "该座位有预约记录，无法删除"

代码实现（AdminController.cs:98-106）：

```csharp
[HttpPost]
public async Task<IActionResult> SeatDelete(int id)
{
    if (!IsLoggedIn()) return RedirectToAction(nameof(Login));
    var seat = await _seatRepository.GetByIdAsync(id);
    if (seat == null) return NotFound();
    await _seatRepository.DeleteAsync(id);
    return RedirectToAction(nameof(SeatIndex));
}
```

SeatRepository 存在 `HasReservationsAsync()` 方法，但 AdminController 未调用。删除有预约记录的座位时，EF Core 外键约束会抛 DbUpdateException，而非返回友好提示。

**影响**：用户体验差（看到 500 错误页面而非"该座位有预约记录，无法删除"提示）。数据库层面不会级联删除（NO ACTION 约束兜底）。

**建议修复**：在 SeatDelete 中增加 `HasReservationsAsync` 校验。

### B-6. 预约管理页和统计页使用内存分页/聚合（P3-记录但不阻塞）

**预约管理页**（AdminController.cs:110-136）：

```csharp
var reservations = await _reservationRepository.GetAllAsync();
// 然后在内存中 Where + OrderBy + Skip + Take
```

**统计页**（AdminController.cs:140-177）：

```csharp
var allReservations = await _reservationRepository.GetAllAsync();
// 然后在内存中 Count + GroupBy
```

docs/09 §7.4 设计为数据库级筛选+分页，docs/09 §8.4 设计为数据库级聚合。当前实现先全量加载到内存再计算。

**影响**：种子数据 0 条预约，当前无性能问题。数据量增大后可能有性能瓶颈。但课堂项目数据量有限，不构成阻塞。

**记录**：已知设计偏差，后续数据量增大时可优化为数据库级查询。

docs/05 A1~A4 设计了管理端独立页面，当前管理端复用用户端 `_Layout.cshtml`，导航栏同时显示用户端和管理端链接。

**影响**：功能不受影响，但管理端界面风格与 docs/05 §3"轻量后台感"的视觉定位有差距。

**记录**：已知 UI 偏差，后续可创建管理端独立 Layout。

### B-8. 统计页热门座位显示 SeatId 而非座位名称（P3-记录但不阻塞）
