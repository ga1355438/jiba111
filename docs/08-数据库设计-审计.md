## B. 发现的问题清单

### B-1：密码存储方式描述不一致（严重度：中）

| 位置 | 描述 |
|------|------|
| PRD-Lite §11 | "密码（MD5 哈希存储）" |
| 系统设计 §2 | "密码加密：MD5" |
| 数据库设计 §4.3 | "密码哈希（PasswordHasher）" |
| 代码实现 | `PasswordHelper.HashPassword()` 使用 `PasswordHasher` |

**问题**：前序文档说 MD5，数据库设计说 PasswordHasher，实际代码已迁移到 PasswordHasher。文档之间不一致。

**建议**：统一为 PasswordHasher，更新 PRD-Lite 和系统设计中的描述。

---

### B-2：时段值命名与实际代码不一致（严重度：低）

| 位置 | 时段值 |
|------|--------|
| 数据库设计 §5.3 | Morning1, Morning2, Afternoon1, Afternoon2, Evening |
| PRD-Lite §11 | "时段，固定 5 个值之一"（未列出具体值） |
| 实际代码 Constants/TimeSlots.cs | Morning1, Morning2, Afternoon1, Afternoon2, Evening |

**问题**：数据库设计中的时段值与代码一致，但 PRD-Lite 未明确列出具体值。

**建议**：在 PRD-Lite §11 中补充时段值定义，或在数据库设计中标注"以代码为准"。

---

### B-3：体验账号数量描述不一致（严重度：低）

| 位置 | 描述 |
|------|------|
| PRD-Lite §14 | "4 个体验账号" |
| PRD-Lite §3 | 张三、李四、王五（3 个学生）+ admin（1 个管理员）= 4 个 |
| 数据库设计 | 未明确列出体验账号数量 |

**问题**：PRD-Lite 说"4 个体验账号"，但实际是 3 个学生账号 + 1 个管理员账号。数据库设计未说明体验账号的处理方式。

**建议**：在数据库设计 §11.2 中明确"体验账号 3 个（张三/李四/王五），通过 Session 模拟，不建用户表"。

---

### B-4：座位列表查询条件描述不准确（严重度：低）

| 位置 | 描述 |
|------|------|
| 数据库设计 §7.1 | "座位列表：Status = 0" |
| PRD-Lite §F02 | "分页展示所有座位及当前状态" |
| 实际代码 | 查询所有座位，不分状态 |

**问题**：数据库设计说座位列表只查 Status=0，但 PRD-Lite 和实际代码是查询所有座位（包括维护中的）。

**建议**：修改为"座位列表：查询所有座位，按 Id ASC 排序"。

---

### B-5：统计口径中"今日预约数"定义需确认（严重度：低）

| 位置 | 描述 |
|------|------|
| 数据库设计 §8.1 | "今日预约数 = ReserveDate = 今天 AND Status ≠ 2" |
| PRD-Lite §12 | "今日预约数 = ReserveDate = 今天且 Status != 2 的记录数" |
| 实际代码 | `r.ReserveDate.Date == today && r.Status != ReservationStatus.Cancelled` |

**问题**：定义一致，但需确认 ReserveDate 的比较方式。数据库设计 §8.4 说"使用 .Date 属性忽略时间部分"，这是正确的。

**建议**：无需修改，但可补充说明"ReserveDate 存储为 datetime2，比较时使用 .Date 属性"。

---

### B-6：缺少数据库版本信息（严重度：低）

| 位置 | 描述 |
|------|------|
| 数据库设计 | 未提及数据库版本 |
| 系统设计 §2 | "SQL Server LocalDB" |

**问题**：未明确 SQL Server LocalDB 的版本，可能影响兼容性。

**建议**：在数据库设计 §1 中补充"SQL Server LocalDB 2019+，兼容 .NET 6"。
